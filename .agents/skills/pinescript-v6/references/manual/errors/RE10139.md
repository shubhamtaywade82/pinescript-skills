# RE10139: Memory Limits Exceeded

### Message
`Memory limits exceeded.`

### Cause
This runtime error occurs when your script exceeds the execution memory threshold allocated by TradingView's servers. 

### Common Causes
1. **Dynamic Collections**: Arrays, maps, or matrices growing too large (e.g. adding elements on every tick/bar without clearing or limiting size).
2. **Infinite Loops or Deep Recursion**: A `while` loop that does not terminate, or a user-defined function calling itself recursively too many times.
3. **Excessive Security Requests**: Running too many or overly large `request.security()` requests, particularly inside dynamic loops.

### ✅ Solutions
- **Limit Array Size**: Use `array.shift()` or `array.remove()` to drop old elements. Keep collections within necessary limits.
- **Verify Loop Conditions**: Ensure all loops have a guaranteed exit condition.
- **Optimize Security Calls**: Do not query requests dynamically. Group security requests into tuples where possible.
