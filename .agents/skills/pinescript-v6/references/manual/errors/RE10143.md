# RE10143: Historical Offset Beyond Buffer Limit

### Message
`The requested historical offset (X) is beyond the historical buffer's limit (Y).`

### Cause
This runtime error occurs when a script references a historical value via the `[]` operator or a built-in function with an offset `X` that is larger than the historical buffer size `Y` currently allocated for that series.

To optimize performance, Pine Script® automatically minimizes buffer size based on static offsets it detects. When you use dynamic variables as index offsets, the compiler cannot predict the buffer size needed.

### ❌ Problematic Code
```pinescript
//@version=6
indicator("RE10143 Demo")

int lookback = input.int(50, "Lookback")
// ❌ Runtime Error: Buffer size not pre-allocated for dynamic lookback index
float pastClose = close[lookback]
plot(pastClose)
```

### ✅ Solution
Use the `max_bars_back()` function on the variable/series to force the compiler to pre-allocate a sufficiently large historical buffer size.

```pinescript
//@version=6
indicator("RE10143 Solution")

int lookback = input.int(50, "Lookback")
// ✅ Correct: Force buffer allocation of at least 100 bars
max_bars_back(close, 100)

float pastClose = close[lookback]
plot(pastClose)
```
